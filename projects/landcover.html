Project 04

Extract High-Resolution Land Cover with GeoAI

This project focuses on high-resolution land cover classification using GeoAI workflows in ArcGIS Pro, followed by vector refinement and green space extraction for interactive visualization in ArcGIS Online. The workflow combines imagery-based classification, post-processing, and thematic filtering to produce a cleaner and more usable land cover result from high-resolution urban imagery.

View Web App Back to Projects

Project Overview

The goal of this workflow was not only to classify land cover from high-resolution imagery, but also to transform the raw model output into a clearer and more interpretable result. After generating the land cover classes, the classified raster was converted into vector features and used to isolate green space categories, creating a more focused layer for analysis and presentation.

This project demonstrates a complete GeoAI workflow: starting from image-based classification, then moving to GIS refinement and layer-based interpretation. For portfolio purposes, the emphasis is placed on software workflow, output quality, and the ability to move from automated classification to a cleaner analytical result.

Input Imagery

The workflow began with a high-resolution orthomosaic used as the base imagery for land cover classification. This imagery provided the visual detail needed to distinguish urban surfaces, vegetation, and water-related areas across the study site.

High-resolution orthomosaic used as the input imagery for classification.

Initial Land Cover Classification

The first result of the GeoAI workflow was a raster-based land cover classification. This raw output included multiple classes such as water, wetlands, tree canopy, shrubland, low vegetation, barren surfaces, structures, impervious surfaces, and impervious roads.

At this stage, the result still represented the direct classified output and was useful for understanding the overall model performance and spatial distribution of classes across the site.

Initial raster output showing the full classified land cover result.

Vector Conversion and Refined Land Cover Layer

After the initial classification, the raster result was converted into vector features. This step made the output easier to manage, symbolize, and reuse in a more analytical GIS workflow. Compared to the raster result, the vectorized layer provided a cleaner structure for class-based interpretation and web presentation.

This stage is important because it shows the transition from model output to a more controlled GIS-ready layer. It reflects not only classification work, but also post-processing and refinement.

Refined land cover features derived from the classified raster output.

Green Space Extraction

A separate green space layer was derived from the classified result by isolating vegetation-related classes. In this project, the extracted green spaces mainly focused on classes such as low vegetation and tree canopy.

This step adds analytical value beyond the original classification. Instead of only presenting all land cover categories, the workflow also produces a targeted thematic output that can support further spatial analysis, communication, and interactive exploration.

Extracted green space layer derived from vegetation-related classes.

Interactive Web Visualization

The final outputs were prepared for interactive visualization in ArcGIS Online. The web app allows the user to explore the processed land cover result, switch between layers, and compare the interpreted output with the original imagery. This final step connects the technical classification workflow with a more accessible and presentation-friendly interface.

Workflow Highlights

  • High-resolution imagery used as the classification input
  • GeoAI-based land cover classification applied to urban imagery
  • Raster output converted into vector features for refinement
  • Green spaces extracted as a focused thematic layer
  • Final result published for interactive web-based exploration
